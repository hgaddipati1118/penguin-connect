# Code of Conduct

## Our Commitment

We want PenguinConnect to be a respectful, technically rigorous, and
privacy-aware open source project. Contributors, maintainers, and community
members are expected to collaborate in good faith and treat each other with
respect.

## Expected Behavior

- be respectful, direct, and constructive
- assume good intent while still challenging technical claims with evidence
- keep review feedback focused on the code, design, tests, docs, and user impact
- protect user privacy when sharing logs, screenshots, or examples
- help keep discussions actionable and on-topic

## Unacceptable Behavior

- harassment, discrimination, or personal attacks
- doxxing or posting private data, including message content, phone numbers,
  email addresses, OAuth secrets, or contact exports
- trolling, deliberate disruption, or bad-faith review behavior
- publishing exploit details or sensitive artifacts before maintainers have a
  chance to respond

## Scope

This code of conduct applies to repository discussions, issues, pull requests,
review comments, and other project-managed communication spaces.

## Enforcement

Project maintainers may edit, remove, or reject comments, commits, issues, or
pull requests that violate this code of conduct. Repeated or severe violations
may result in temporary or permanent bans from project spaces.

If you experience or witness unacceptable behavior, open a minimal issue asking
for maintainer contact or use the repository security reporting path when the
incident involves privacy or security-sensitive material.
